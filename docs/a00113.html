PCL_EXPORT FILE* pcl_popen ( const tchar_t *  command,
  ... 
)

#include <pcl/stdio.h>

‪Pipe a stream to or from a process.

This differs from the POSIX popen in that there is no type argument. Instead, this using symbols (like Perl open) to indicate redirection and optional flags as the first set of characters within the command arguments.

Mode Description Same as...
< read from stdout popen("cmd", "r")
> write to stdin popen("cmd", "w")
^ read from stderr [no equivalent, ^ used to mimic FISH shell]
The below documents the other "flags" that can be specified in mode:
- set FD_CLOEXEC popen("cmd", "re" OR "we") <- since glibc 2.9
! set O_NONBLOCK [no equivalent]
// read from the process's stdout without blocking. Process created with
// the following command: `ls -l /home/username`
tchar_t *homedir = pcl_getenv("HOME");
FILE *stream = pcl_popen(_T(">! ls -l %ts"), homedir);
pcl_free(homedir);
Parameters
command‪pointer to the command and optional flags
...‪variable arguments
Returns
‪stream pointer or NULL on error

◆ pcl_fdopen()

PCL_EXPORT FILE* pcl_fdopen ( pcl_file_t *  file)

#include <pcl/stdio.h>

‪Open a file handle as a stream.

On success, a file stream is returned that must be closed via fclose. The provided file handle is released and cannot be used again. However, if this function fails, the caller is still repsonsible for closing the given file handle.

pcl_file_t *file = get_the_file();
FILE *fp = pcl_fdopen(file);
if(fp) // success
{
fprintf(fp, "'fp' is valid here! 'file' is not.\n");
fclose(fp);
}
else // failure
{
printf("'file' is valid here. 'fp' is not.\n");
pcl_close(file); // callee still responsible for this...
}
Parameters
file‪pointer to a file handle
Returns
‪a file stream that must be closed with fclose

◆ pcl_tmpfile()

PCL_EXPORT FILE* pcl_tmpfile ( const tchar_t *  dir,
const tchar_t *  prefix,
const tchar_t *  suffix,
tchar_t **  pathp,
bool  keep 
)

#include <pcl/stdio.h>

‪Create and open a unique temporary file.

This function will make several attempts to create the unique temporary file. The below is a list (in order) of what directories are attempted:

  1. ‪try the dir argument
  2. ‪Windows TMP or Unix TMPDIR
  3. ‪Windows TEMP or Unix P_tmpdir
  4. "."
    Parameters
    dir‪Optional directory path to create temp file within, can be NULL.
    prefix‪Optional prefix for the generated pathname. Can be NULL
    suffix‪Optional suffix for the generated pathname. Can be NULL
    pathp‪pointer to a tchar_t string that will receive the temp pathname. This can be NULL. When not NULL, this must be freed by the caller.
    keep‪If false, returned temp file will be automatically deleted once closed. When true, the file will persist after file close.
    Returns
