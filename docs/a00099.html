Detailed Description

The buffer API provides a dynamically sized buffer for putting text and/or binary data.

The buffer supports two modes: binary and text. In either mode, a trailing NUL is always written at the end of the buffer. This space is reserved internally.

Binary Mode

In binary mode, the buffer is an array of data type: integers, strings and blobs. Strings are written to the buffer with trailing NULs, avoiding the need to prefix string data with a length.

pcl_buf_t *buf = pcl_buf_init(NULL, 64, false);
pcl_buf_putstr(buf, "Hello"); // buffer contents: "Hello\0\0"
pcl_buf_putstr(buf, "World"); // buffer contents: "Hello\0World\0\0"
// binary mode only, "Hello\0World\0[32 bits in network-byte order]\0"
pcl_buf_putint32(buf, 3897463623);
pcl_buf_free(buf);

Binary mode supports putting and getting integer types: 8-bit, 16-bit, 32-bit and 64-bit. In all cases, integer types are written to the buffer in network-byte order.

Text Mode

In text mode, the buffer is an array of bytes: string buffer. In this mode, integer types cannot be used and those functions will return an error. Strings are written without a trailing NUL byte.

pcl_buf_t buf; // use stack memory
pcl_buf_init(&buf, 64, true);
pcl_buf_putstr(&buf, "Hello"); // buffer contents: "Hello\0"
pcl_buf_putstr(&buf, "World"); // buffer contains: "HelloWorld\0"
int len;
char *text = pcl_buf_data(&buf, &len); // len is optional
printf("%d] %s\n", len, text); // 10] HelloWorld
pcl_buf_clear(&buf); // 'text' invalid

Once all text data has been put, the data buffer can be retreived using pcl_buf_data. The returned pointer is a NUL terminated string. It is very common to use the printf style buffer functions while in text mode but can also be used in binary.
