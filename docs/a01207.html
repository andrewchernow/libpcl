◆ pcl_json_decode()

PCL_EXPORT pcl_json_t* pcl_json_decode ( const char *  json,
size_t  len,
const char **  end 
)

#include <pcl/json.h>

‪Decode a JSON string.

This parses a JSON string that represents a single JSON value. A JSON value can be an object {}, array [], string "", number -102.84, boolean true or false or a null null. Parsing stops when the parsing of the JSON value at json is complete, regardless of the len argument. However, if a complete JSON value is not found before reaching json + len, an error occurs. The json string can contain bogus data after the value being parsed, since the parser stops at the end of the value. For example:

{
"name": "Henry Ford"
}#invalid json here

The above is completely valid, since a valid JSON object exists and ends before any invalid JSON text is encountered. When the end argument is supplied, it would be pointing at the '#' in the above example. This is very useful if a string contains multiple JSON values. pcl_json_decode can be called a multiple times passing the value of end as json, allowing one to iterate through a series of json values.

Parameters
json‪pointer to a json string. This must be NUL terminated if len is 0.
len‪number of bytes within json argument. If 0, json must be NUL terminated.
end‪pointer to the first character not parsed. This can be NULL.
Returns
‪pointer to a json value or NULL on error

It is safe to use the hash table module for managing a JSON object. Internally, a json object is backed by a pcl_htable_t. For a given pcl_json_t, ensure the type member is 'o' and then use json->object with the htable functions. However, do not use pcl_htable_free.

◆ pcl_json_string()

PCL_EXPORT pcl_json_t* pcl_json_string ( char *  str,
uint32_t  flags 
)

#include <pcl/json.h>

‪Create a string.

Parameters
str‪pointer to a NUL-terminated string
flags‪a bitmask of flags. If PCL_JSON_SKIPUTF8CHK is set, no UTF-8 validation is performed. If PCL_JSON_SHALLOW is set, a direct pointer to str is stored: ie. no copy. If PCL_JSON_SHALLOW is not set, str is copied. Note that if PCL_JSON_EMPTYASNULL and PCL_JSON_SHALLOW are both set and str is empty, caller is still repsonsible to free the empty string. This is because the json module isn't managing it. For this reason, one should avoid setting both of these flags. If PCL_JSON_EMPTYASNULL, then an empty str will cause a json null object to be returned. If PCL_JSON_ALLOWNULL is set, if str is NULL, a json null object will be returned.
Returns
‪pointer to a json object or NULL on error. Note that a NULL str is an error unless PCL_JSON_ALLOWNULL is set.
See also
‪pcl_json_string_len
