Detailed Description

The tchar_t functions map char or wchar_t string functions.

They are all implemented as macros but documented as functions. This makes the docs as clear as possible in regards to return values, parameters, etc. These functions (macros) will map wchar_t versions on Windows and char versions on Unixes. The wchar_t versions are UTF-16 while the char versions are expected to be UTF-8. The tchar_t versions are prefixed with tcs or include a t in the name (like pcl_totupper). The tcs prefix extends the wcs naming.

◆ pcl_tcsfbytes()

int pcl_tcsfbytes ( tchar_t *  buf,
size_t  bufl,
uint64_t  nbytes,
int  decimal 
)

#include <pcl/string.h>

‪Format a number in byte units.

This uses multiples of 1024 and supports the following units: Kibibyte, Mebibyte, Gibibyte, Tebibyte, Pebibyte, Exibyte. The formatted unit is appened to the converted number value as a single uppercase character (no spaces): 10K, 50M, etc. For values less than 1024, no unit is appened; this is just bytes.

Parameters
buf‪pointer to a buffer to write formatted value
bufl‪character length of buf
nbytes‪number of bytes, value to format
decimal‪number of decimal places to include in the formatted value.
Returns
‪number of character written to buf or -1 on error.
Note
‪implemented as a macro
