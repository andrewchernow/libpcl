115 /** Decode a JSON string. This parses a JSON string that represents a single JSON value. A JSON
116  * value can be an object \c {}, array \c [], string \c "", number \c -102.84, boolean \c true or
117  * \c false or a null \c null. Parsing stops when the parsing of the JSON value at \a json
118  * is complete, regardless of the \a len argument. However, if a complete JSON value is not found
119  * before reaching \a json \c + \a len, an error occurs. The \a json string can contain bogus data
120  * after the value being parsed, since the parser stops at the end of the value. For example:
121  * @code
122  * {
123  * "name": "Henry Ford"
124  * }#invalid json here
125  * @endcode
126  * The above is completely valid, since a valid JSON object exists and ends before any invalid
127  * JSON text is encountered. When the \a end argument is supplied, it would be pointing at
128  * the '#' in the above example. This is very useful if a string contains multiple JSON values.
129  * pcl_json_decode can be called a multiple times passing the value of \a end as \a json,
130  * allowing one to iterate through a series of json values.
131  *
132  * @param json pointer to a json string. This must be NUL terminated if \a len is 0.
133  * @param len number of bytes within \a json argument. If 0, \a json must be NUL terminated.
134  * @param end pointer to the first character not parsed. This can be \c NULL.
135  * @return pointer to a json value or \c NULL on error
136  */
137 PCL_EXPORT pcl_json_t *pcl_json_decode(const char *json, size_t len, const char **end);

177 /** Create a string.
178  * @param str pointer to a NUL-terminated string
179  * @param flags a bitmask of flags. If ::PCL_JSON_SKIPUTF8CHK is set, no UTF-8 validation
180  * is performed. If ::PCL_JSON_SHALLOW is set, a direct pointer to \a str is stored: ie. no copy.
181  * If ::PCL_JSON_SHALLOW is not set, \a str is copied. Note that if ::PCL_JSON_EMPTYASNULL and
182  * ::PCL_JSON_SHALLOW are both set and \a str is empty, caller is still repsonsible to free the
183  * empty string. This is because the json module isn't managing it. For this reason, one should
184  * avoid setting both of these flags. If ::PCL_JSON_EMPTYASNULL, then an empty \a str
185  * will cause a json null object to be returned. If ::PCL_JSON_ALLOWNULL is set, if \a str
186  * is \c NULL, a json null object will be returned.
187  * @return pointer to a json object or \c NULL on error. Note that a \c NULL \a str is an error
188  * unless ::PCL_JSON_ALLOWNULL is set.
189  * @see pcl_json_string_len
190  */
