‪Pause execution of the current thread for the given number of nanoseconds.

Parameters
nsecs‪nanoseconds to sleep, guarenteed to sleep for this amount when cond parameter is NOT provided.
cond‪When not NULL, this is called every cond_interval seconds and aborts the sleep operation if it returns true.
cond_interval‪nanosecond interval between calls to optional "cond" argument. If this is zero, the default interval of 10000000 ns is used (10 ms). Note: cond is called prior to the first sleep. This argument is ignored if cond is NULL.
Returns
‪true for success and false otherwise. When cond is not NULL, true can mean the cond returned true or the function slept for the entire nsecs.
