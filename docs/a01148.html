◆ key_len

size_t key_len

‪Key length in bytes.

The default is zero, which means variable-length (strings). When implementing key_equals and hashcode callbacks, this value is not used by the table. When using the default key_equals, strcmp is used when this is zero and memcmp when non-zero. When using the default hashcode, strlen is used when this is zero to compute the key_len otherwise this value is used directly.

The table should be empty when setting this value: after pcl_htable_create or pcl_htable_clear.

◆ min_loadfac

float min_loadfac

‪The minimun load factor of the table.

When the number of entries in the table falls below the product of this value and current capacity, the table is shrunk to half its capacity rounded up to the nearest prime. The default value is 0.20f.

The table should be empty when setting this value: after pcl_htable_create or pcl_htable_clear.

◆ max_loadfac

float max_loadfac

‪The maximum load factor of the table.

When the number of entries in the table exceeds the product of this value and current capacity, the table is grown to double its capacity rounded up to the nearest prime. The default is 0.75f.

The table should be empty when setting this value: after pcl_htable_create or pcl_htable_clear.

void(* remove_entry) (const void *key, void *value, void *userp)

‪Called when an entry is being removed from the table.

There is no default version, this is set to NULL on the hash table. When NULL, only the entry object itself is freed. In most cases, an application will want to set this to free keys and/or values.

◆ hashcode

uintptr_t(* hashcode) (const void *key, size_t key_len, void *userp)

‪Compute a hash code for the given key.

Google's farmhash is used as the default hashfunc for both 32 and 64-bit architectures. On 32-bit machines, a 32-bit code is computed and on 64-bit machines, a 64-bit hash code.

Hash code values use the uintptr_t data type. This means the number of bits allowed depends on the number of bits within a uintptr_t.

Parameters
key‪pointer to the key to hash
key_len‪key length in bytes
userp‪pointer to optional user data. This can be NULL.
Returns
‪hash code no more than unitptr_t bytes
