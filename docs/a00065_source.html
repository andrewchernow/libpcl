60 /** Parse a JSON string. This parses a JSON string that represents a JSON value. A JSON value
61  * can be an object \c {}, array \c [], string \c "", number \c -102.84, boolean \c true or
62  * \c false or a null value \c null. Parsing stops when the parsing of the JSON value at \a json
63  * is complete, regardless of the \a len argument. However, if a complete JSON object is not found
64  * between \a json \c + \a len, an error occurs. The \a json string can contain bogus data
65  * after the value being parsed, since the parser stops at the end of the value. For example:
66  * @code
67  * {
68  * "name": "Henry Ford"
69  * }#invalid json here
70  * @endcode
71  * The above is completely valid, since a valid JSON object exists and ends before any invalid
72  * JSON text is encountered. When the \a end argument is supplied, it would be pointing at
73  * the '#' in the above example. This is very useful if a string contains multiple JSON values.
74  * pcl_json_parse can be called a multiple times passing the value of \a end as \a json.
75  *
76  * @param json pointer to a json string. This must be NUL terminated if \a len is 0.
77  * @param len number of bytes within \a json argument. If 0, \a json must be NUL terminated.
78  * @param pointer to the first character not parsed. This can be \c NULL.
79  * @return pointer to a json value or \c NULL on error
80  */
81 PCL_EXPORT pcl_json_value_t *pcl_json_parse(const char *json, size_t len, const char **end);
