◆ entries

pcl_htable_entry_t* entries

‪A sequential array of entries as they are inserted.

This not allows for preserving insertion order, by avoids allocating each individual entry. This always contains capacity available entries. The next insert index is bound to count_used, not count. count_used is always incremented, unlike count. It is reset to 0 by pcl_htable_clear and reset to count during an internal rehash.

Note
‪each entry element is part of a collision list by using its next member to indicate the index of the next entry in the list. A next value of -1 indicates the end of the list.
‪allocated in same block of memeory as entry_lookup

◆ entry_lookup

int* entry_lookup

‪An entry lookup array where each element is an entries index value.

When given a key, the key's hashcode is bitwise AND'd with the table's capacity minus 1, which produces the index to this array; termed the hashed index or hashidx. Elements in this array are set to -1 to indicate they are not in use. This always contains capacity indexes.

Note
‪allocated in same block of memeory as entries

◆ key_len

size_t key_len

‪Key length in bytes.

The default is zero, which means variable-length (strings). This value is only used by the default key_equals and hashcode callbacks. If a custom version of both callbacks are set, this can remain unset unless the custom implementation requires this value.

The table should be empty when setting this value: after pcl_htable or pcl_htable_clear.

◆ min_loadfac

float min_loadfac

‪The minimun load factor of the table.

When the number of entries in the table falls below the product of this value and current capacity, the table is shrunk to half its capacity (always a power of 2). The default value is 0.20f.

The table should be empty when setting this value: after pcl_htable or pcl_htable_clear.

◆ max_loadfac

float max_loadfac

‪The maximum load factor of the table.

When the number of entries in the table exceeds the product of this value and current capacity, the table is grown to double its capacity (always a power of 2). The default is 0.75f.

The table should be empty when setting this value: after pcl_htable or pcl_htable_clear.

◆ remove_entry

void(* remove_entry) (const void *key, void *value)

‪Called when an entry is being removed from the table.

There is no default version, this is set to NULL on the hash table. When NULL, the key and value are not freed which might be the desired behavior. In most cases, an application will want to set this to free keys and/or values. In either case, the entry itself is marked as deleted (NULL key).

◆ hashcode

uintptr_t(* hashcode) (const void *key, size_t key_len)

‪Compute a hash code for the given key.

Google's farmhash is used as the default hashfunc for both 32 and 64-bit architectures. On 32-bit machines, a 32-bit code is computed and on 64-bit machines, a 64-bit hash code.

For the default hashcode implementation, if key_len is zero, it is assumed that key is a string and strlen is used to get the key_len. The key_len passed is the pcl_htable_t.key_len member.

This is only executed once for each entry put into the table. The computed hash code is cached within the entry object. This is also executed, on the key argument, each time either pcl_htable_get, pcl_htable_lookup or pcl_htable_remove are called.

Hash code values use the uintptr_t data type. This means the number of bits allowed depends on the number of bits within a uintptr_t.

Parameters
key‪pointer to the key to hash
key_len‪key length in bytes
Returns
‪hash code no more than unitptr_t bytes
